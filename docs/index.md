# Software Factory Security Framework (SF²)

Demand for security work grows faster than your team's capacity to deliver it, and there is a day when the two cross. SF² is a strategic response. Instead of racing to hire, it asks where your organization actually stands, then sequences the security work that compounds from there.

The framework rests on two ideas. Five Universal Security Conditions that every software-producing organization owes, in priority order: Supply Chain, Third-Party, Process, Runtime, and Adaptive Capacity. And four strategic positions, set by your Blast Radius and Operational Readiness, that decide what to fund first and how fast to move.
